server.js

Accept onLog callback in scrapeAll(). Log source switches, rate limits, retries, per-subreddit results.
redditBulk.js

youtubeSearch.js

Full browser-based scraping with visible window. Puppeteer as fallback when APIs/HTTP fail.

2a Puppeteer Config Section
Add PUPPETEER config: headless (default true), windowWidth (400), windowHeight (300), timeout (15s).

Before vs After

Before (Original)

  • Progress bars only — no detail
  • Hidden headless browser
  • Flat 2s sleep on errors
  • One failed source = total loss
  • Duplicate sentiment in 3 files
  • Dead code never used
  • YouTube: single HTML regex approach
  • No caching — re-scrapes everything
  • Report names collide
  • No graceful shutdown
  • Magic numbers in 4+ files

After (All Phases)

  • Live terminal log — every action visible
  • Visible browser window (400×300)
  • Exponential backoff + jitter + retry
  • 3-tier fallback + Puppeteer fallback
  • Centralized sentiment analysis
  • All dead code integrated as fallbacks
  • YouTube: yt-dlp + HTML + Puppeteer
  • 24h file-based caching
  • Timestamped filenames
  • SIGTERM/SIGINT handlers
  • All config in config.js

Web Application

How Users Run It

The project is already a self-contained web application. No cloud, no API keys, no cost.

$ git clone <repo>
$ cd research-toolkit
$ npm install
$ npm run web
# opens http://localhost:3000

The web UI includes: topic input, depth slider, Reddit/YouTube toggles, real-time progress, report viewer, and history.
